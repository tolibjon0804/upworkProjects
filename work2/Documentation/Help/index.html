Template Structure

SPOT is based on a grid system from Twitter Bootstrap 3, for more info about how the grid system works please check:http://getbootstrap.com/css/#grid

index.html is the core file of the template but as you can see there is no index.html file. SPOT comes with 14 predefined variants and when you will choose the variant you want to work on then just simpy delete the number form the file name. More about variants later in the decumentation. 

Folder Structure

  • css - all css files and fonts are placed here.
  • images - all images including mockup images are plaecd here.
  • js - Java Script files included with the template.
  • php - PHP files included with the template. Files related to newsletter form and contact form.
  • video - video files included with the template - for video background version.

 

 

All in one homepage

SPOT comes with All in One approuch. This applies only to homepage section. It means that all variants of homepage are placed in html file but you control witch ones are displayed by typing a class in class parameter. For example if you would want to have a video in the background of homepage you should add a "background_video" class. This approach makes editing very easy but adds one additional step to the process of editing. After you are done customising you should delete unused elements from html file (for faster site load).

How To: Icons

SPOT comes with more than 1500 icons. Those are vector icons merged into a icon font so you can easly use them.

Below is a simple markup for an icon:

<div data-icon="&#xe0f3;"></div>

It is a simple div with data-icon atribute representing an icon. You can check out what icons are included, just open Documentation/rapid-icon-font/icons-reference.html file. Under each icon you can find it code to insert as data-icon value. 

All icons are vector based and look beautiful on retina displays :)

How To: HiDpi Images

SPOT looks beautiful on retina display and other HiDpi displays. It uses retina.js to check if user is viewing James on retina display and then checks if there are "retina ready" images - equivalent to standard images, and if yes then it will display bigger images insted of standard ones.

If you want to use retina.js just make sure to do 2x bigger images with @2x on the end of the file name, before the extension, and put them on the server. 

 

Retina.js will work only when your website is on web server, it will not work locally.

How To: Helper Classes

With James you can use (add a class to your object) some helper classes, listed below:

  • .align-center - aligns the elements to center
  • .align-left - aligns elements to the left
  • .align-right - aligns elements to the right
  • .img-circle - closes the image in a circular shape(use on <img>, to achive a circle use square image)
  • .img_responsive - makes your image responsive
  • .img-rounded_corner - adds rounded corners to your image

How To: Homepage elements and backgrounds

SPOT comes with several variants of homepage, all of them you can activate by adding/removing corresponding class in:

<section class="hero_fullscreen background_single menu_bar-waypoint" data-animate-down="menu_bar-hide" data-animate-up="menu_bar-hide" >

Below you will find a list of classes you can use here. Remember that classes thet are bolded in the above code need to stay unchanged.

  • gradient overlay - adds a gradient to homepage background - works only when background_slider or background_video is active
  • background_solid - fills homepage background with a solid color (defined in your selected color css file) with predesigned mockup images, with this modyficator you szhould add another one selecting one of two predefined mockups - solid_01 or solid_02.
  • background_single - single image in the background
  • background_slider - fade in/out image slider in the background
  • background_video - adds video in the homepage background

So if you would like to slider in a background with gradient overlay applied to it your code would look like this:

<section class="hero_fullscreen background_slider gradient_overlay menu_bar-waypoint" data-animate-down="menu_bar-hide" data-animate-up="menu_bar-hide" >

Changing the speed of scroll animation

Menu links are captured by a script that makes the scroll to target section smooth and with some easing. You can change the animation parameters in main.js file, find this code:

menuItems.click(function(e){
        var href = $(this).attr("href"),
        offsetTop = href === "#" ? 0 : $(href).offset().top-navbarheight+2;
        $('html, body').stop().animate({ scrollTop: offsetTop  }, 1000, 'easeInOutExpo');
        e.preventDefault();
    });

bolded number is a time to scroll in ms, bolded word next to it is a type of easign used. More on easing types here: http://jqueryui.com/easing/

 

 

 

 

 

 

 

How To: Animations

As you can see when you scroll the page first time elements are smoothy animating into positions. Each animated element has an additional class to determine if the element is animation and the order of animation.

As default there is 10 animation elements per section and the class name is determined by section id and _animX (X is 1-10) representing the order of animation. That way each section is independent and you can rearange them as you like. So if you want to set animations in for example #more_info section, all you need to do is add classes more_info_animX to selected elements you would like to animate in order of animation.

You can also control the animation type and time offset. To do so open main.js file and find Elements_animation function. Here is a part of this function code:

     $('#about_video').waypoint(function() {
            setTimeout(function(){$('.about_video_anim1').addClass('animated fadeInUp')},0);
            setTimeout(function(){$('.about_video_anim2').addClass('animated fadeInUp')},200);
            setTimeout(function(){$('.about_video_anim3').addClass('animated fadeInUp')},400);
            setTimeout(function(){$('.about_video_anim4').addClass('animated fadeInUp')},600);
            setTimeout(function(){$('.about_video_anim5').addClass('animated fadeInUp')},800);
            setTimeout(function(){$('.about_video_anim6').addClass('animated fadeInUp')},1000);
            setTimeout(function(){$('.about_video_anim7').addClass('animated fadeInUp')},1200);
            setTimeout(function(){$('.about_video_anim8').addClass('animated fadeInUp')},1400);
            setTimeout(function(){$('.about_video_anim9').addClass('animated fadeInUp')},1600);
            setTimeout(function(){$('.about_video_anim10').addClass('animated fadeInUp')},2000);
        }, { offset: '50%' });

 

First bolded line representas a place that you need to scroll to launch the animation. In our case it will be section with id "about_video". It needs to be scrolled by 50% to run the animation (offset parameter). Second bolded line presents a single element that is animated. First you declare the element class (.about_video_anim1), then you can declare the type on fanimation you want to have (fadeInUp) and next you declare the delay (0) in ms. For animation types you can use all animations presented here: http://daneden.github.io/animate.css/  

Homepage animations are operated by preloader. You can edit them in main.css file.
